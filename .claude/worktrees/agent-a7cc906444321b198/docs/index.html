A Claude Code status line that shows token totals and dollar cost.

Claude Code reports rate-limit percentages. statusline-tokens adds the numbers underneath: raw tokens and API-priced cost for the rolling 5-hour and 7-day windows, plus current context size. One PowerShell script, no runtime.

Windows / PowerShell 5.1+ / MIT

Why

Claude Code reports rate-limit percentages but no raw token counts. Other status-line packages mirror the percentages. None tell you what a session is costing.

Install

Two files, one restart. Works in Windows Terminal, the VS Code integrated terminal, and any 256-color ANSI host.

  1. Copy the script.

    Place it in %USERPROFILE%\.claude\, next to your settings.json.

    Copy-Item statusline-tokens.ps1 "$env:USERPROFILE\.claude\"

  2. Wire it up.

    Add (or replace) the statusLine block in settings.json.

    {
  "statusLine": {
    "type": "command",
    "command": "powershell -NoProfile -File %USERPROFILE%\\.claude\\statusline-tokens.ps1",
    "padding": 0
  }
}

  3. Restart Claude Code.

    The bar appears on the next prompt cycle. Cold render is ~1.0s, warm is ~0.3s (20-second JSON cache).

Methodology

For every .jsonl line carrying a "usage" block in ~/.claude/projects/**/*, the script does the following.

Percentages are not computed locally. They come straight from rate_limits.five_hour.used_percentage and its 7-day sibling — the same authoritative numbers Claude Code uses internally for throttling decisions.

Token totals are summed locally, the same way ccusage does it: a recursive scan of every transcript, regex-extracted, deduplicated by message.id, windowed by ISO timestamp.

Cost is computed per turn using the model family detected on that transcript line (Opus, Sonnet, or Haiku) and the matching input, output, and cache rates. Ephemeral cache creation bills at the 5-minute rate when the breakdown isn't exposed — the API default for unflagged blocks, and the cheaper of the two tiers.

  1. Parse timestamp.Skip the line if it's outside the 7-day cutoff.
  2. Dedupe by message.id.Assistant turns log once per content block; raw summation would triple-count.
  3. Sum the four fields.input + output + cache_creation + cache_read.
  4. Price it.Detect model family on this transcript line; apply the matching API rate.
  5. Window the totals.Add to 7d. If the timestamp also clears the 5-hour cutoff, add to 5h.

Field-level regex extraction is ~10× faster than ConvertFrom-Json per line on a 20 MB transcript pile. A 20-second JSON cache keeps the bar snappy on every re-render.
