Token counts and dollar cost in your Claude Code status line.

Anthropic's status line shows rate-limit percentages. statusline-tokens adds the numbers underneath — raw tokens and API-priced cost for the rolling 5-hour and 7-day windows.

What it adds

The number you actually need is missing.

Claude Code reports rate-limit percentages. Other status-line packages mirror them. None of them tell you what the session is costing you.

$

API-rate cost

Dollar cost for both windows, priced per model family with the correct input, output, and cache rates.

#

Raw token totals

Actual tokens consumed in the 5-hour and 7-day windows, summed locally from your transcripts.

Headroom

The number changes color before you hit a wall.

The 5h and 7d percentages come straight from your signed-in Claude account — the same number Claude Code uses to throttle you. The status line tints it so you read the band, not the digits.

Want different bands? Set STATUSLINE_PCT_THRESHOLDS to a warn,crit pair before Claude Code starts — e.g. STATUSLINE_PCT_THRESHOLDS=65,85 shifts yellow to 65% and red to 85%. The defaults are 50,80.

Install

Three commands. Roughly thirty seconds.

First time touching Claude Code? Follow the guided path — setup.ps1 does the file copy, the JSON merge, and a smoke test for you. No Node, no npm, no extra tools.

Before you start
  • Claude Code is installed and you've had at least one conversation in it. That conversation creates the ~/.claude/ folder we'll be writing into.
  • PowerShell 7+ recommended. Open a terminal and run pwsh -v. If it errors, run winget install Microsoft.PowerShell. Windows PowerShell 5.1 (the one preinstalled on Windows 10/11) also works.
  • Git on your PATH. If git --version works in PowerShell, you're set. Otherwise grab it from git-scm.com.
  1. Step 01

    Clone the repo

    Anywhere on disk — the cloned folder is throwaway, setup.ps1 copies the files it needs out of it.

    git clone https://github.com/Gabriel-Dalton/`
  claude-statusline-tokens
  2. Step 02

    Run the setup script

    Copies the scripts to ~/.claude/, shows the JSON it wants to add to settings.json, and asks Y/n before it writes. A backup is saved first.

    cd claude-statusline-tokens
.\setup.ps1

    Tip Curious before committing? Run .\setup.ps1 -DryRun first — it prints the exact diff without touching anything.

  3. Step 03

    Open a new Claude Code window

    The status line shows up on the next prompt. The current conversation won't pick up the change — you need a fresh one for the new settings.json to load.

    # first render
cold  ~1.0s   # PS startup + scan
warm  ~0.3s   # cached, <20s old
Prefer to wire things up by hand?

If you'd rather edit settings.json yourself, the manual path lives in the README's manual install section. It's three steps: copy the .ps1 next to settings.json, paste a statusLine block in, restart.

On macOS or Linux?

The scripts run under pwsh (PowerShell 7+) on Unix too. Same three commands — just install pwsh first via your package manager (brew install powershell on macOS, your distro's repo on Linux), then clone and run ./setup.ps1.

Something went wrong?

The README's troubleshooting section covers the three or four ways this can fail: execution-policy blocks, mojibake characters, empty ~/.claude/projects/, and a malformed settings.json falling through silently.

Methodology

How the numbers are computed.

For every .jsonl line that carries a "usage" block in ~/.claude/projects/**/*, the script applies the following procedure.

The percentages are not computed locally — they come straight from rate_limits.five_hour.used_percentage and its 7-day sibling. These are the same authoritative numbers Claude Code uses internally for throttling decisions.

The token totals beneath them are summed locally, exactly the way ccusage does it: a recursive scan of every transcript, regex-extracted, deduplicated by message.id, windowed by ISO timestamp.

Cost is computed per turn, using the actual model family detected on that transcript line — Opus, Sonnet, or Haiku — and the correct input, output, and cache rates. Ephemeral cache creation bills at the 5-minute rate when the breakdown isn't exposed — the API default for unflagged blocks, and the cheaper of the two tiers.

  1. 01 Parse timestamp Skip the line if it's outside the 7-day cutoff.
  2. 02 Dedupe by message.id Assistant turns log once per content block; raw summation would triple-count.
  3. 03 Sum the four fields input + output + cache_creation + cache_read.
  4. 04 Price it Detect model family on this transcript line; apply the matching API rate.
  5. 05 Window the totals Add to 7d. If the timestamp also clears the 5-hour cutoff, add to 5h.
Field-level regex extraction is ~10× faster than ConvertFrom-Json per line on a 20 MB transcript pile. A 20-second JSON cache keeps the bar snappy on every re-render.
