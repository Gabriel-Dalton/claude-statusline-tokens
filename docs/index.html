Built carefully

Six design decisions that matter.

Native percentages

The 5h and 7d % come straight from rate_limits.*.used_percentage — the same numbers Claude Code uses internally for throttling.

Real token totals

Token sums are computed from ~/.claude/projects/**/*.jsonl, exactly the way ccusage does it — but without leaving your terminal.

Deduped by message id

Assistant turns log once per content block. Naive summing triples the number. We dedupe by msg_id so the totals match reality.

API-rate costing

Per-turn cost using the actual model family (Opus / Sonnet / Haiku) and the right input, output, cache-read, and cache-write rates.

20-second cache

First render scans ~20MB of transcripts in under a second. Subsequent renders hit a JSON cache so the bar refreshes instantly.

One file, no deps

A single statusline-tokens.ps1. No npm, no install steps beyond a copy + a settings.json edit. Customize colors and segments in place.

Setup

Install in 30 seconds.

Two files, one restart. Works with Windows Terminal, VS Code, and any 256-color ANSI terminal.

Copy the script

Drop statusline-tokens.ps1 into your .claude folder.

# PowerShell
Copy-Item statusline-tokens.ps1 `
  "$env:USERPROFILE\.claude\"

Wire it up

Add this statusLine block to ~/.claude/settings.json:

{
  "statusLine": {
    "type": "command",
    "command": "powershell -NoProfile -ExecutionPolicy Bypass -File \"%USERPROFILE%\\.claude\\statusline-tokens.ps1\"",
    "padding": 0
  }
}

Restart Claude Code

Or just start a new conversation. The bar appears on the next prompt cycle. No npm, no service, no daemon.

Internals

How it counts tokens.

For every .jsonl line that carries a "usage" block:

  1. Parse the timestamp. Skip the line if it's outside the 7-day window.
  2. Dedupe by message.id. Assistant turns log once per content block (thinking, tool_use, …) and each entry repeats the same usage. Counting them all triples the number.
  3. Sum the four fields: input_tokens + output_tokens + cache_creation_input_tokens + cache_read_input_tokens.
  4. Price it. Detect the model family from the transcript and bill input, output, cache-read, and cache-write at the right rate (5m vs 1h ephemeral cache).
  5. Add to 7d. If the timestamp is also inside the 5-hour cutoff, add to 5h.

Regex extraction beats ConvertFrom-Json per line by ~10× on a 20MB transcript pile. A 20-second JSON cache (~/.claude/statusline-tokens.cache.json) keeps the bar snappy on every re-render.
